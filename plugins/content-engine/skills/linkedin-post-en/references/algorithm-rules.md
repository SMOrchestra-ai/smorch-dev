# LinkedIn Algorithm Rules — English

## The Fold
LinkedIn shows ~3 lines before "see more."
Hook must work completely within first 3 lines.
Test: Cover everything below line 3. Still compelling? If not — rewrite.

## Line Length
Write for mobile: 5-10 words per line for hooks and punch lines.
Max 15 words for explanation lines.

## Whitespace
Empty line between every block. Non-negotiable.

## Format
- Numbered sections: `1/` only
- Sub-bullets: `→` only
- No bold except sub-headers
- No italic, no ALL CAPS

## Engagement
- Comments: P.S. question drives comments. Easy to answer, personally relevant.
- Saves: Lists and frameworks get saved.
- Reposts: Contrarian hooks + specific proof drive reposts.
- DMs: `DM me "[KEYWORD]" — I'll send you [specific deliverable].`

## Character Limit
3000 characters max. Run count_chars.py before delivering.
If over: cut least essential section first. Never cut hook, proof, CTA, P.S.
